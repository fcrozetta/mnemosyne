# EPIC-00 · 0.1.0 (Alpha) Release Train

**Suggested milestone:** `0.1.0 (Alpha)`  
**Suggested labels:** `type:epic`, `type:release`, `prio:p0`, `phase:0`

## Outcome

Define, track, and protect the 0.1.0 alpha release train so the project ships on **2026-04-24**.

## Why this exists

Without an explicit release train, Mnemosyne will sprawl into architecture theater and miss the first public cut.

## Scope

- lock release date
- lock product cut
- maintain board/checklist
- control scope drift
- decide green / amber / red at each phase boundary

## Phases

### Phase 0
- release brief
- board
- labels taxonomy
- epic tracker files

### Phase 5
- final release checklist
- changelog / notes
- tag decision
- publish decision

## Lanes

### Lane A · Planning
- [ ] release brief approved
- [ ] board approved
- [ ] milestone name frozen

### Lane B · Scope control
- [ ] non-goals documented
- [ ] optional work explicitly cut
- [ ] no surprise workstreams added

### Lane C · Release readiness
- [ ] release gates checked
- [ ] blocker list empty or accepted
- [ ] tag naming decision made

## Exit criteria

- release date remains 2026-04-24
- all other epics exist and are linked from the board
- no unresolved ambiguity on what 0.1.0 alpha means

## Risks

> [!WARNING]
> The fastest way to miss this release is to sneak in UI work, auth work, or Ananke/Moirai coupling under the excuse of “just preparing the future.”
